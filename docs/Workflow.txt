# CTT Engine — New Computer Setup Workflow

## 1. Clone the Repository
```bash
git clone git@github.com:tl-uk/CTT.git
cd CTT

## 2. Install System Dependencies (macOS)
# Apple Silicon note: This project requires native arm64 libraries. Homebrew on Apple Silicon installs to /opt/homebrew, 
# which the CMakeLists.txt auto-detects.
```bash
# Install core build tools
brew install cmake ninja pkg-config

# Install project libraries (ZeroMQ is required before CMake configure)
brew install zeromq

# Install everything else declared in the repo Brewfile
brew bundle

——————

## 3. Build the C++ Engine
# Standard Build (Daily Use)
```bash
cmake -B build -G Ninja
cmake --build build

# Clean Build (First Time or After CMake Changes)
# Required flags for modern CMake compatibility: Fetched dependencies (cppzmq, flecs) contain legacy 
# cmake_minimum_required declarations. Without the policy override, CMake 3.31+ hard-fails.
```bash
rm -rf build
cmake -B build -G Ninja \
  -DCMAKE_BUILD_TYPE=Release \
  -DCMAKE_OSX_ARCHITECTURES=arm64 \
  -DCMAKE_POLICY_VERSION_MINIMUM=3.5
cmake --build build

# Verify the Binary
```bash
# Check that it exists
ls -l build/CTT_Engine

# Verify architecture (should say 'arm64', not 'x86_64')
file build/CTT_Engine

# Quick smoke test
./build/CTT_Engine

————————

## 4. Set Up Python Environment
# Recommended: Use uv for fast, deterministic Python management.
```bash
# Create venv with native arm64 Python 3.13
uv venv --python 3.13
source .venv/bin/activate

# Install Python dependencies
uv pip install -r requirements.txt

# Python Dependencies (If Not in requirements.txt)
# If the project needs SUMO or ZeroMQ bindings outside the requirements file:
```bash
uv pip install bptk-py pyzmq eclipse-sumo

| Task                     | Command                                                                                                                                                            
| -----------------------  | ------------------------------------------------------------------------------------------------------------------------------------------------

# Running locally hosted Flecs
# Go to any directory (your home folder is fine)
cd ~

# Clone the explorer (one-time only)
git clone https://github.com/flecs-hub/explorer.git

# Enter the folder that contains the static files (in a new terminal)
cd explorer/etc

# Start a local HTTP server
python3 -m http.server 8000

# Open Chrome and go to:
http://localhost:8000

# The local Explorer page will automatically detect and connect to CTT_Engine's REST API on localhost:27750.
